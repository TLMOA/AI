# AI模块综合执行方案（融合 NiFi、联邦学习 与 双模部署）

- 版本：v3.0
- 日期：2026-04-22

## 目标
整合 NiFi 数据处理、联邦学习框架与公有/私有双模部署战略，形成一套清晰、可执行、兼顾效率与隐私的端到端数据交付与 AI 赋能框架。

---

## 一、总体结论与战略框架

### 核心目标
提供一套兼顾“公有化”与“私有化”部署场景的可执行框架，明确不同场景下的技术路径（静默转换、直接传输、联邦学习），并配套必要的安全、合规与运维体系，确保项目可落地、可信任且可持续。

### 部署模式与建议
- 公有化部署：工厂数据托管于我方数据库，优先采用中心化训练以提升建模效率与迭代速度（在法务/合规授权下）。
- 私有化部署：工厂数据驻留在工厂侧数据库，优先采用联邦学习以保障“数据不出域”；当联邦效果不足时，采用混合策略（高敏数据走联邦，低敏数据中心化）。

---

## 二、静默转换（共享基础能力）

### 定义
静默转换指在不改变工厂业务感知的前提下，按计划自动将数据库内容导出为标准化产物（CSV、NDJSON 等），用于训练或下游处理。

### 原则与要点
- 自动化与可审计：任务配置化、全过程记录并可回溯。 
- 可回退：支持快速切回本地处理路径。 
- 最小权限：仅导出必要字段，支持字段级脱敏和聚合。 
- 原子写入：临时文件 → 完整性校验 → 原子重命名。 
- Manifest：每个产物附带 manifest.json（schema、行数、checksum、timestamp、rule_version）。
- 契约测试：Local 与 NiFi 路径输出行为通过契约测试确保一致性。

---

## 三、私有化场景：两种静默传输方式对比

### 方式 A：直接拉取（中心化收集）
- 流程：工厂静默导出文件，通过安全通道（SFTP/API/签名 URL）传回中心存储；中心进行数据清洗、统一建模。
- 优点：实现简单、训练效率高、便于统一数据治理与特征工程。 
- 风险：数据出域带来合规与信任问题，需合同、脱敏及访问控制保障。 
- 适用场景：合规允许且工厂授权共享详细数据的场景（或非敏感场景）。

### 方式 B：联邦学习（FL）
- 流程：中心下发训练任务与全局模型；工厂端的 Client Agent 在本地读取数据并运行 `client_train()`；仅上报模型参数/差分与 summary；中心端聚合更新。 
- 优点：数据不出域、合规门槛低、用户信任高，适合跨工厂协同建模。 
- 风险：Non-IID 导致收敛/性能问题；通信与调度复杂；参数泄露风险需引入 Secure Aggregation/DP。 
- 适用场景：工厂坚持数据不可出域或受严格监管（金融、医疗等）。

## 四、融合架构概览

设计理念：统一编排、双模执行、数据不动计算动。

核心组件：
- 前端（UI/JetLinks 适配层）
- 统一编排层（FastAPI）：任务管理、节点与证书治理、模型仓库、审计。 
- 数据流中枢（NiFi / Exporter）：静默转换、manifest 与 DLQ。 
- 训练层：中心训练服务（用于方式A/公有化）与 Flower Server（用于方式B/联邦）。
- Client Agent（Sidecar）：读取本地产物、执行训练或上报文件、管理证书与心跳。 
- 模型仓库：版本、checksum、签名 URL 与审计记录。

图示（逻辑示意）：
- FastAPI 作为控制面，统一下发任务并协调 NiFi / Client Agent 与训练层。
- 数据在 Local/NiFi 两条路径生成同一逻辑目录产物，Client Agent 根据执行模式（直接上传或本地训练）采取不同动作。

## 五、实施分阶段计划（建议）

### 阶段一：V0.1 - 基础闭环与直接拉取验证（第1–3周）
- 完成 FastAPI 核心接口（任务、文件、模型）、契约与 client_train() 接口规范。 
- 实现 Local Executor（本地静默转换）并产出带 manifest 的标准文件。 
- 实现 Client Agent 的文件传输能力（SFTP/签名 URL 上传）并完成中心存储对接。 
- 演示中心化训练闭环与验收（契约测试通过）。

### 阶段二：V0.2 - 联邦 PoC 与 NiFi 生产接入（第4–6周）
- 部署 Flower Server 并与 FastAPI 集成；在实验室用 2–3 个客户端跑 5–10 轮联邦训练，验证无原始数据外传。 
- 部署 NiFi 流程并做双路径一致性测试（Local vs NiFi 输出一致性 ≥ 99.9%）；固化契约测试并完成回滚演练。 
- 配置 mTLS、证书注册与基础监控面板。 

### 阶段三：V0.3 - 生产就绪、混合策略与生态融合（第7–8周及以后）
- 实现存储抽象（Local ↔ Ceph S3），支持双写与迁移。 
- 在 FastAPI 中支持基于策略选择训练模式（混合策略）。 
- 引入 Secure Aggregation / 差分隐私（可配置），完善治理平台与证书管理。 
- 准备 JetLinks 集成（前端适配层、API 契约对齐）。

## 六、关键保障体系

### 安全与合规
- 传输：全链路 TLS/mTLS，企业 PKI 管理证书。 
- 准入治理：节点证书指纹、白名单、任务签名、证书撤销（CRL/OCSP）。 
- 隐私增强：短期采用基础安全聚合，中长期引入差分隐私（DP）与参数化噪声。 
- 审计：记录每次导出/文件传输/联邦轮次的 traceId、参与方、样本量/参数量、model_hash、时间戳（保存 ≥ 90 天）。

### 运维与可观测
- 指标：导出成功率、导出延时、客户端心跳率、联邦轮次完成数、模型性能（F1/LOSS）、DLQ 大小。 
- 告警与 Runbook：为导出失败、证书过期、节点掉线、文件传输中断等场景预设恢复和回滚流程。 
- 契约测试：数据契约、接口契约与双路径一致性纳入 CI，作为 PR 阻断项。

## 八、成功标准
- 路径完备：系统支持“公有化/直接拉取+中心训练”与“私有化/静默转换+联邦训练”两条核心路径。 
- 隐私可信：能证明并验证联邦训练过程中“原始数据零出域”；对于直接拉取，能提供完整合规使用链路。 
- 生产稳健：具备双路径容灾、可观测性、应急回滚机制与治理能力。
