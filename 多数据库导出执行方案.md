# 多数据库导出 - 执行方案

**目标**
在现有“数据库导出（MySQL）”栏顶部增加一个数据库类型下拉（`db_type`），仅在该栏内显示并生效。用户在该栏选择的数据库类型将用于该栏的所有导出操作（手动触发与定时任务），导出文件仍写入现有 `NIFI_OUTPUT_DIR` 的对应子目录（例如 `output_csv` / `output_json` / `output_tsv`），与当前 MySQL 行为一致。

---

## 范围与边界
- 前端只修改 `数据库导出（MySQL）` 栏（不修改全局定时任务面板）。
- 后端按 `ExportJobModel.db_config.type` 执行导出；短期优先保证 MySQL 行为不变并新增对 PostgreSQL 的支持（视需要）。
- 导出文件位置：写入 `NIFI_OUTPUT_DIR` 的格式目录，不自动强制镜像到 `in_data`。

---

## 假设
- 已有 MySQL 导出在本系统运行良好（现有行为保持不变）。
- 若支持新数据库，运行环境可安装额外驱动（例如 `psycopg2`）。

---

## 交付物
- 前端改动：`v1-frontend` 中 `数据库导出（MySQL）` 栏顶部添加 `db_type` 下拉并按需显示额外字段，提交时把 `db_config.type` 一并发送。文件：`v1-frontend/app.js`（或具体负责导出表单的文件）。
- 后端变更（如需）：`v1-backend` 中 `engine_from_config` / `run_export_job` 能识别 `db_config.type` 并构建对应 engine（至少支持 `postgres` 作为 MVP）。文件：`v1-backend/app/export_worker.py`、`v1-backend/app/engine_factory.py`。
- 测试：最少一套 E2E 验证（Postgres 或另一 DB + MySQL 回归）。
- 文档：本执行方案与依赖清单（驱动包、配置示例）。

---

## 详细任务分解（前端优先）
1. 前端：在 MySQL 导出栏顶部增加 `db_type` 下拉
   - 修改文件：`v1-frontend/app.js`（或导出表单所在文件）
   - 下拉项：`MySQL`, `PostgreSQL`, `SQLServer`, `SQLite`, `Oracle`
   - 交互：当选择非-MySQL（如 PostgreSQL）时显示额外连接字段（如 `port`、`database`、可选 `dsn`）。
   - 提交：创建/编辑导出任务时，`db_config` 中包含 `type` 字段。
   - 最小变更原则：对 MySQL 选择时 UI 完全不变。

2. 前端：构造并发送示例请求体
   - 示例请求体（创建任务）：
```
{
  "job_name": "导出-sensor_data-postgres",
  "db_config": {
    "type": "postgres",
    "host": "db.example.local",
    "port": 5432,
    "user": "test",
    "password": "***",
    "database": "metrics",
    "table": "sensor_data"
  },
  "file_format": "csv",
  "destination": { "type": "local", "path": "nifi_data/output_csv" }
}
```

3. 后端（按需）
   - 在 `engine_from_config` 中根据 `db_config.type` 生成 SQLAlchemy URL 或使用对应驱动。
   - 若缺少驱动，记录依赖：`psycopg2-binary`（Postgres）、`pyodbc`（SQLServer）、`cx_Oracle`（Oracle）。
   - 确保 `run_export_job` 使用 `db_config` 构建 engine 并执行现有增量导出逻辑，输出路径保持和 MySQL 一致（写入 `NIFI_OUTPUT_DIR/output_*`）。

4. 测试
   - 单元测试：验证前端构造的 `db_config`（含 `type`）与后端解析逻辑。                 
   - 集成/E2E：至少执行两轮：
     a. MySQL 回归：提交 MySQL 导出任务，手动触发并验证 `NIFI_OUTPUT_DIR/output_csv` 文件生成与审计记录。  
     b. PostgreSQL（或目标 DB）：提交 Postgres 导出任务，手动触发并验证 `NIFI_OUTPUT_DIR/output_csv` 生成与审计记录。

5. 文档与运维
   - 在 `README.md` 或部署文档中加入：额外驱动依赖与安装、`NIFI_OUTPUT_DIR` 路径说明、凭证及秘钥管理建议。

---

## 验收标准（验收测试要点）
- UI：`数据库导出（MySQL）` 栏顶部可选择 `db_type`，选择 MySQL 时界面无变化。
- 数据契约：创建的任务包含 `db_config.type` 并存入 DB。
- 执行：手动触发或调度触发均使用任务内 `db_config` 执行导出，导出文件写入 `NIFI_OUTPUT_DIR/<format>`。
- 审计：`audit.log` 与 `factory_reports.ndjson` 记录包含导出状态、行数与目标路径（含 `factory_id`）。
- 安全：测试期间不在日志或非加密存储中泄露明文凭证。
