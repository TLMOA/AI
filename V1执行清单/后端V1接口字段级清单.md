# 后端V1接口字段级清单

## 1. 目标与范围

本清单用于V1阶段后端封装落地，覆盖你负责的能力：

- 任务管理
- 文件管理
- 标签管理
- 调度管理
- NiFi集成状态回填

不包含真实大模型训练后端能力，仅保留占位字段与扩展点。

## 2. 统一约定

### 2.1 基础响应结构

成功响应：

```json
{
  "code": 0,
  "message": "OK",
  "data": {},
  "traceId": "uuid"
}
```

失败响应：

```json
{
  "code": 1001201,
  "message": "validation failed",
  "data": null,
  "traceId": "uuid"
}
```

### 2.2 状态枚举

- jobStatus: `PENDING | RUNNING | SUCCEEDED | FAILED | CANCELED`
- jobType: `CONVERT | TAG_MANUAL | TAG_AUTO | COPY_MULTI_FORMAT | IMPORT_EXTERNAL`
- scheduleStatus: `ENABLED | DISABLED`

### 2.3 追踪与幂等

- Header: `X-Trace-Id`（可选，后端可自动生成）
- Header: `X-Idempotency-Key`（创建任务时建议必填）

## 3. 任务管理接口

## 3.1 创建任务

- Method: `POST`
- Path: `/api/v1/jobs`

请求体：

```json
{
  "jobType": "CONVERT",
  "source": {
    "sourceType": "MYSQL_TABLE",
    "dbSourceId": "jetlinks-mysql-main",
    "tableName": "sensor_data",
    "where": ""
  },
  "target": {
    "format": "CSV",
    "outputDir": "/data/jobs"
  },
  "tagConfig": {
    "mode": "AUTO",
    "ruleId": "NIFI_RULE_ID_V5"
  },
  "copyFormats": ["JSON", "TSV"],
  "cron": "",
  "runBy": "user-001",
  "remark": "manual trigger"
}
```

字段说明：

- `jobType`：任务类型。
- `source.sourceType`：`MYSQL_TABLE | FILE_UPLOAD`。
- `target.format`：`CSV | JSON | TSV`。
- `tagConfig.mode`：`MANUAL | AUTO`。
- `copyFormats`：多格式副本目标。

返回：

```json
{
  "jobId": "job_20260330_001",
  "status": "PENDING",
  "createdAt": "2026-03-30T14:00:00+08:00"
}
```

## 3.2 任务详情

- Method: `GET`
- Path: `/api/v1/jobs/{jobId}`

返回字段：

- `jobId`
- `jobType`
- `status`
- `progress`（0-100）
- `source`
- `target`
- `errorCode`
- `errorMessage`
- `nifiFlowId`
- `createdAt`
- `startedAt`
- `finishedAt`

## 3.3 任务列表

- Method: `GET`
- Path: `/api/v1/jobs`

查询参数：

- `status`（可选）
- `jobType`（可选）
- `pageNo`（默认1）
- `pageSize`（默认20）
- `createdFrom`（可选）
- `createdTo`（可选）

## 3.4 取消任务

- Method: `POST`
- Path: `/api/v1/jobs/{jobId}/cancel`

返回字段：

- `jobId`
- `status`（应转为`CANCELED`）
- `canceledAt`

## 4. 文件管理接口

## 4.1 查询任务产物

- Method: `GET`
- Path: `/api/v1/jobs/{jobId}/outputs`

返回字段：

- `fileId`
- `fileName`
- `fileFormat`
- `fileSize`
- `storageType`（V1为`LOCAL`）
- `storagePath`
- `createdAt`

## 4.2 文件下载

- Method: `GET`
- Path: `/api/v1/files/{fileId}/download`

说明：

- V1本地文件直出下载。
- 后续Ceph切换后可改为签名URL重定向。

## 4.3 文件预览

- Method: `GET`
- Path: `/api/v1/files/{fileId}/preview`

查询参数：

- `offset`（默认0）
- `limit`（默认100）

返回字段：

- `columns`
- `rows`
- `total`

## 5. 标签管理接口

## 5.1 手动标签提交

- Method: `POST`
- Path: `/api/v1/tags/manual`

请求体：

```json
{
  "fileId": "file_001",
  "records": [
    { "rowId": "12", "label": "FAULT" },
    { "rowId": "13", "label": "NORMAL" }
  ],
  "operator": "user-001"
}
```

## 5.2 自动标签任务触发

- Method: `POST`
- Path: `/api/v1/tags/auto`

请求体：

```json
{
  "fileId": "file_001",
  "ruleId": "NIFI_RULE_ID_V5",
  "outputFormat": "CSV",
  "operator": "user-001"
}
```

返回：

- `jobId`
- `status`

## 5.3 标签规则列表

- Method: `GET`
- Path: `/api/v1/tags/rules`

返回字段：

- `ruleId`
- `ruleName`
- `ruleVersion`
- `enabled`
- `description`

## 6. 调度管理接口

## 6.1 创建调度

- Method: `POST`
- Path: `/api/v1/schedules`

请求体：

```json
{
  "name": "sensor_data_auto_convert",
  "jobTemplate": {
    "jobType": "CONVERT",
    "source": { "sourceType": "MYSQL_TABLE", "tableName": "sensor_data" },
    "target": { "format": "CSV" }
  },
  "cron": "0 */30 * * * ?",
  "visibility": "USER_VISIBLE"
}
```

## 6.2 调度列表

- Method: `GET`
- Path: `/api/v1/schedules`

## 6.3 更新调度状态

- Method: `PATCH`
- Path: `/api/v1/schedules/{scheduleId}`

请求体：

```json
{
  "status": "DISABLED"
}
```

## 6.4 删除调度

- Method: `DELETE`
- Path: `/api/v1/schedules/{scheduleId}`

## 7. 数据表建议

## 7.1 ai_job

- `id` bigint pk
- `job_id` varchar(64) unique
- `job_type` varchar(32)
- `status` varchar(16)
- `progress` int
- `source_json` text
- `target_json` text
- `error_code` varchar(32)
- `error_message` varchar(1024)
- `nifi_flow_id` varchar(128)
- `idempotency_key` varchar(128)
- `created_by` varchar(64)
- `created_at` datetime
- `started_at` datetime
- `finished_at` datetime

## 7.2 ai_file_asset

- `id` bigint pk
- `file_id` varchar(64) unique
- `job_id` varchar(64)
- `file_name` varchar(255)
- `file_format` varchar(16)
- `file_size` bigint
- `storage_type` varchar(16)
- `storage_path` varchar(1024)
- `created_at` datetime

## 7.3 ai_schedule

- `id` bigint pk
- `schedule_id` varchar(64) unique
- `name` varchar(128)
- `cron_expr` varchar(64)
- `status` varchar(16)
- `visibility` varchar(32)
- `job_template_json` text
- `created_by` varchar(64)
- `created_at` datetime
- `updated_at` datetime

## 8. 开发顺序（建议）

1. 先落 `POST /jobs` + `GET /jobs/{jobId}`。
2. 再接入NiFi状态回填和失败错误码映射。
3. 再落文件预览/下载与标签接口。
4. 最后补调度接口与幂等机制。

## 9. 验收最小集

1. 手动创建转换任务成功并完成。
2. 可查询任务状态从PENDING到SUCCEEDED。
3. 可查看并下载输出文件。
4. 自动标签任务可触发并产出标签文件。
5. 定时任务可创建、停用、恢复。
