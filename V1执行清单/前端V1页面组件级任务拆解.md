# 前端V1页面组件级任务拆解（简化可改造版）

## 1. 前端策略

V1阶段前端先独立可运行，不依赖JetLinks联调结果。

实现原则：

1. 页面层只依赖前端API适配层。
2. JetLinks相关能力通过特性开关控制（默认关闭）。
3. 页面配置化，便于后续快速改造。

## 2. 路由清单

建议路由：

- `/jobs` 任务中心
- `/jobs/:id` 任务详情
- `/files` 文件中心
- `/tags` 标签中心
- `/schedules` 调度中心
- `/integration` 集成状态（占位）

## 3. 页面与组件拆解

## 3.1 任务中心 `/jobs`

组件：

1. `JobCreateDrawer`
   - 任务类型选择
   - 数据源配置
   - 输出格式配置
   - 一键提交
2. `JobFilterBar`
   - 按状态、类型、日期筛选
3. `JobTable`
   - 展示jobId、类型、状态、进度、创建时间
4. `JobActionButtons`
   - 查看详情
   - 取消任务
   - 重试任务（可选）

交互：

- 页面加载拉取任务列表。
- 创建任务后自动跳详情页。

## 3.2 任务详情 `/jobs/:id`

组件：

1. `JobStatusCard`
   - 当前状态、进度、错误摘要
2. `JobStepTimeline`
   - 创建、执行、完成节点
3. `JobParamsPanel`
   - 输入参数回显
4. `JobOutputList`
   - 输出文件列表与下载入口
5. `JobLogPanel`（可选）
   - 错误日志与traceId

交互：

- 状态轮询间隔3-5秒，完成后停止。

## 3.3 文件中心 `/files`

组件：

1. `FileFilterBar`
   - 文件格式、日期、任务来源
2. `FileTable`
   - 文件名、格式、大小、创建时间
3. `FilePreviewModal`
   - CSV/JSON表格预览
4. `FileDownloadButton`
   - 文件下载

交互：

- 点击预览调用preview接口，分页展示。

## 3.4 标签中心 `/tags`

组件：

1. `TagModeSwitcher`
   - 手动标签/自动标签切换
2. `ManualTagEditor`
   - 选择文件
   - 行级标签编辑
   - 提交
3. `AutoTagForm`
   - 选择文件
   - 选择规则
   - 触发自动标签任务
4. `TagTaskResult`
   - 展示标签任务状态与输出文件

交互：

- 自动标签触发后跳转任务详情。

## 3.5 调度中心 `/schedules`

组件：

1. `ScheduleCreateDrawer`
   - 名称、cron、模板任务配置
2. `ScheduleTable`
   - 展示状态、最近执行时间
3. `ScheduleActionButtons`
   - 启停
   - 编辑
   - 删除

交互：

- 创建后立即可见并支持启停。

## 3.6 集成状态 `/integration`（占位页）

组件：

1. `IntegrationStatusCard`
   - JetLinks联调状态：未接入/联调中/已接入
2. `FeatureTogglePanel`
   - JetLinks相关入口开关
3. `TodoListCard`
   - 待联调事项列表

## 4. 前端API适配层

建议目录：

- `src/services/api/client.ts`
- `src/services/api/jobApi.ts`
- `src/services/api/fileApi.ts`
- `src/services/api/tagApi.ts`
- `src/services/api/scheduleApi.ts`
- `src/services/adapters/backendAdapter.ts`
- `src/services/adapters/jetlinksAdapter.ts`（V1占位）

要求：

1. 页面只调用adapter暴露的方法。
2. 后续JetLinks接入只替换adapter实现。

## 5. 状态管理建议

建议最小store切片：

- `jobStore`：列表、详情、轮询状态
- `fileStore`：文件列表、预览数据
- `tagStore`：标签规则、提交状态
- `scheduleStore`：调度列表与启停状态
- `featureStore`：特性开关

## 6. 特性开关清单

默认关闭：

- `feature.jetlinksAlarmPush`
- `feature.jetlinksSso`
- `feature.jetlinksWorkbenchEntry`

默认开启：

- `feature.localTaskFlow`
- `feature.localFilePreview`
- `feature.localTagging`

## 7. 视觉与交互优先级

P0（必须）：

1. 任务创建与列表
2. 任务详情与进度
3. 文件预览与下载
4. 标签手动/自动入口
5. 调度创建与启停

P1（可后补）：

1. 批量操作
2. 高级筛选
3. 图表看板

## 8. 开发节奏（10个工作日）

1. Day1-2：路由骨架 + API适配层 + 任务中心。
2. Day3-4：任务详情 + 状态轮询。
3. Day5-6：文件中心 + 预览下载。
4. Day7-8：标签中心（手动+自动）。
5. Day9：调度中心。
6. Day10：联调、修复、验收演示脚本。

## 9. 验收最小集

1. 前端可创建转换任务并看到进度变化。
2. 前端可查看产物并下载。
3. 前端可触发自动标签并查看结果。
4. 前端可创建和启停调度任务。
5. JetLinks开关默认关闭时，页面流程不受影响。
