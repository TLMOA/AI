# Day1详细任务单（V1启动日）

在1个工作日内完成“后端任务主链路 + 前端基础骨架 + 联调最小闭环”的可运行版本。

## 2. 时间分块执行（建议）

## 09:30 - 10:30 任务启动与环境确认

1. 确认后端项目运行方式与端口。
2. 确认前端项目运行方式与端口。
3. 确认NiFi容器在线与目标流程可见。
4. 约定traceId与错误码规范（先写入项目README或开发约定文档）。

产出：

- 运行环境记录
- 端口与基础配置清单

## 10:30 - 12:00 后端接口最小集（第一批）

开发接口：

1. POST /api/v1/jobs
2. GET /api/v1/jobs/{jobId}
3. GET /api/v1/jobs

要求：

1. 接口先返回可测试结构，不必一次接完NiFi。
2. 状态机按PENDING/RUNNING/SUCCEEDED/FAILED定义。
3. 每个响应包含traceId。

产出：

- 第一批接口可通过Postman调用
- 示例请求与响应

## 13:30 - 15:00 后端接入NiFi触发（最小联通）

1. 在POST /jobs中调用NiFi触发逻辑（先接1条已验证流程）。
2. 在GET /jobs/{jobId}中回填NiFi运行状态。
3. 失败时统一错误映射（先做3类：参数错误、流程触发失败、运行失败）。

产出：

- 任务创建可驱动NiFi执行
- 任务状态可查询变化

## 15:00 - 16:30 前端骨架与任务页（第一批）

页面：

1. /jobs 任务列表
2. /jobs/:id 任务详情

组件最小集：

1. 创建任务弹窗
2. 列表表格
3. 状态标签与进度条
4. 详情页基础信息卡片

产出：

- 前端可创建任务并跳转详情页

## 16:30 - 17:30 前后端联调（最小闭环）

1. 前端调用POST /jobs。
2. 轮询GET /jobs/{jobId}。
3. 看到状态从PENDING进入RUNNING（如能到SUCCEEDED更好）。

产出：

- 联调截图或录屏
- 问题清单

## 17:30 - 18:00 收尾与日报预填

1. 整理未完成项。

## 3. 完成判定（Day1）

满足以下3项即Day1达标：

1. 前端可发起任务。
2. 后端可创建并查询任务状态。
3. 至少1条任务能触发NiFi流程执行。

## 4. 风险与预案

1. 风险：NiFi接口鉴权问题。
   - 预案：先用后端本地模拟状态，鉴权打通后替换。
2. 风险：前端接口字段频繁变更。
   - 预案：前端统一通过adapter层消费数据。
3. 风险：状态查询抖动。
   - 预案：前端轮询容错，后端返回最近稳定状态。
